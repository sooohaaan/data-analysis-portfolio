## 2. 목표

### 비즈니스 목표
- 앱스토어 평균 별점 **2.56점 → 3.1점 이상** 달성 (6개월, PTT blueplus+ 수준)

### 제품 목표

- **결제·환불·지갑(11.0%)** 해결 → 결제 성공률 98% 이상

## 범위 외 (Non-Goals)

> 본 PRD는 KOKKOK 슈퍼앱(모빌리티+쇼핑+EV충전) 중 **충전 파트**에 한정된다.
> 맥락: KOKKOK Move(라오스 현지 운영 EV 라이드헤일링) → KOKKOK 슈퍼앱으로 확장, 우리 팀은 **충전 파트** 담당.
> 아래 항목은 본 PRD가 명시적으로 **다루지 않는다**.

| # | 범위 외 항목 | 소관 / 사유 |
|---|------------|-----------|
| 1 | 라이드헤일링 배차·드라이버 매칭 로직 | KOKKOK Move 기존 모빌리티 팀 소관 |
| 2 | 쇼핑 모듈 | 슈퍼앱 별도 파트 |
| 3 | 충전기 하드웨어 제조·펌웨어 개발 | 제조사(Costel 등) 협업 영역 — 앱은 OCPP 표준 연동만 담당 |
| 4 | 결제 PG/지갑 코어 인프라 신규 구축 | 기존 KOKKOK Wallet 활용 (본 PRD는 충전 연동·UX 레이어만) |
| 5 | 라오스 외 국가 확장 | 1차 범위는 라오스 비엔티안·사반나켓 (타 국가는 후속 단계) |

> ℹ️ Feature 3(OCPP 준수 검증)·Feature 10(OTA 펌웨어)은 하드웨어 표준 연동 범위에 한해 다루며, 펌웨어 자체 개발은 위 항목 3에 따라 범위 외이다.

## 3. 핵심 기능 명세

### 🔴 P0 — 즉시 해결

#### Feature 1: 충전 세션 자동 복구 (Auto-Recovery)

**근거**: 충전앱 레이어 최대 Pain Point — **앱 불안정·작동불가 27.6%** (부정 리뷰 1,227건 중 최다)

**문제**: 앱 충돌·네트워크 끊김 시 충전 세션이 끊기고 결제만 완료됨

**솔루션**:
```
충전 시작 → 앱 종료 / 네트워크 단절 / 통신 오류
     ↓
서버에 세션 상태 저장 (heartbeat 30초)
     ↓
앱 재실행 시 → 진행 중인 세션 자동 감지
     ↓
"충전이 진행 중입니다. 이어서 진행하시겠습니까?" 팝업
```

**검증 지표**: 세션 복구 성공률 ≥ 95% / 재연결 시간 ≤ 10초

**✅ 수용 기준 (Acceptance Criteria)**:
- 크래시-프리 세션율 ≥ 99%
- 세션 끊김 시 30초 내 자동 복구 성공률 ≥ 95%
- 결제 완료·충전 미개시 상태로 종료된 세션 0건 (모두 복구 또는 자동 환불로 정산)

---

#### Feature 2: 간편 온보딩 + KYC/로그인 (외국인 친화) 〔NEW〕

**근거**: 충전앱 레이어 2위 Pain Point — **계정·인증·KYC 15.0%**. 가입·로그인·본인인증 단계 이탈이 충전 시작 자체를 막음. 특히 외국인 온보딩 마찰(영어 리뷰 Negative 65.8%)이 큼.

**솔루션**:
- 소셜 로그인(Apple/Google) + 전화번호 OTP 단일 단계 로그인
- KYC 단계 분리: 가입·첫 충전은 즉시, 본인인증은 결제 한도 상향 시점으로 지연
- 외국인 여권 기반 KYC 플로우 + 영어 안내 전면 제공

**✅ 수용 기준 (Acceptance Criteria)**:
- 신규 가입 → 첫 충전 완료까지 중단율 ≤ 15%
- 외국인(여권) 인증 성공률 ≥ 90%
- 소셜 로그인 + OTP 단일 단계 로그인 성공률 ≥ 98%

---

#### Feature 3: OCPP 2.0.1 표준 준수 검증 (HW 협업)

**근거**: 상호운용성 표준 확보 요건 — 멀티벤더 충전기 환경에서 앱-충전기 통신 일관성을 보장하기 위한 기술 표준. (충전앱 VOC가 직접 뒷받침하는 인과 근거는 아님)

**솔루션**:
- Costel 등 HW 제조사와 OCPP 2.0.1 준수 협약
- 충전기-앱 Handshake 오류 로그 수집 → 원인 분리
- 표준 비준수 충전기 교체 우선순위 목록 관리

**구현 주체**: HW 제조사(Costel) 협업 필수

**✅ 수용 기준 (Acceptance Criteria)**:
- 연동 충전기의 OCPP 2.0.1 핵심 메시지(BootNotification·StartTransaction·StopTransaction) 준수율 100%
- 앱-충전기 Handshake 오류율 ≤ 1%

### 🟠 P1 — 단기 개선

#### Feature 4: 지갑·결제 안정화 + 오프라인 RFID 결제

**근거**: **결제·환불·지갑 11.0%** (충전앱 레이어 3위 Pain Point) / PTT blueplus+는 앱+카드 이중 지원으로 만족도 우위

**솔루션**:
- 지갑 잔액·결제·환불 상태 동기화 안정화 (이중 차감·환불 누락 방지)
- RFID 카드 태깅으로 사전 등록 크레딧 차감 (네트워크 불필요)
- 라오스·베트남 현지 편의점 카드 충전 연계

**✅ 수용 기준 (Acceptance Criteria)**:
- 결제 실패율 ≤ 2%
- 결제 후 충전 미개시 시 자동 환불 100%
- 이중 차감·환불 누락 분쟁 0건

---

#### Feature 5: 충전 시작 전 결제 성공 확인 화면

**근거**: 결제·환불·지갑 11.0% 중 "결제됐는데 충전이 시작 안 됨 — 환불도 안 됨" 반복 패턴

**솔루션**:
```
결제 완료 → [결제 확인 화면 3초] "결제 완료 ✓ | 충전 시작 중..."
         → 충전 시작 신호 수신 → [충전 진행 화면] kWh·금액 실시간 표시
```

**✅ 수용 기준 (Acceptance Criteria)**:
- 결제 완료 후 충전 시작 신호 수신 표시까지 ≤ 5초
- "결제됐는데 충전 미개시" 패턴 CS 티켓 베이스라인 대비 ≥ 80% 감소

---

#### Feature 6: 외국인 여행자 다국어 UI + 국제 결제 지원

**근거**: 외국인·다국어 **1.5%** + 영어 리뷰 Negative **65.8%** (영어 리뷰 빈출: "payment failed", "no foreign card", "no English")

**솔루션**:
- 앱 내 영어·한국어·태국어 다국어 지원
- Visa/Mastercard 해외 카드 직접 등록 및 결제
- 여행자 1회성 QR 결제 (앱 가입 없이 충전 가능)

**✅ 수용 기준 (Acceptance Criteria)**:
- 해외 카드(Visa/Mastercard) 등록·결제 성공률 ≥ 90%
- 영어 UI 전체 화면 커버리지 100% (미번역 문자열 0건)
- 여행자 1회성 QR 결제 충전 완료율 ≥ 85%

### 🟡 P2 — 중기 개선

#### Feature 7: 충전소 실시간 위치·세션 상태 지도

**근거**: **충전소·위치 2.4%** (충전소를 찾지 못하거나 사용 가능 여부·세션 상태를 알 수 없음)

**솔루션**:
- 지도에서 충전기 가용 상태 실시간 표시 (available/in-use/error)
- 진행 중 충전 세션 상태(kWh·잔여시간) 지도/홈에서 확인
- 이동 중 충전기 사전 예약 + 도착 예상 시간 기반 추천

**✅ 수용 기준 (Acceptance Criteria)**:
- 지도 충전소 상태(available/in-use/error) 실시간성 ≤ 60초 지연
- 충전기 가용 상태 표시 정확도 ≥ 95%

---

#### Feature 8: QR 스캔 실패 자동 재시도 + 수동 ID 입력

**근거**: 설치·업데이트(6.6%)·앱 안정성(27.6%)에 수반되는 충전 시작 마찰의 보조 완화책. (단독 근거 비중은 낮아 P2로 배치)

**솔루션**:
```
QR 스캔 실패 → 자동 재시도 (3초) → 재실패 → 충전기 ID 6자리 수동 입력
```

**✅ 수용 기준 (Acceptance Criteria)**:
- 수동 ID 입력 fallback 충전 시작 성공률 ≥ 90%
- 자동 재시도(3초) 후 QR 스캔 최종 성공률 ≥ 85%

---

#### Feature 9: 설치·업데이트 안정화

**근거**: **설치·업데이트 6.6%** — 업데이트 후 실행 불가·강제 업데이트 마찰

**솔루션**:
- 하위 호환 강제 업데이트 정책 정비 (필수 업데이트 최소화)
- 업데이트 실패 시 롤백·재시도 안내
- 설치 용량·다운로드 단계 최적화

**✅ 수용 기준 (Acceptance Criteria)**:
- 업데이트 직후 크래시율 베이스라인 대비 증가 0 (업데이트가 안정성을 악화시키지 않음)
- 강제(필수) 업데이트 단계 이탈률 ≤ 5%
- 업데이트 실패 시 롤백·재시도 안내 노출률 100%

---

#### Feature 10: 충전기 펌웨어 원격 업데이트 (OTA)

**근거**: 멀티벤더 충전기 펌웨어 표준화·무중단 패치를 위한 운영 기반 (앱 안정성 27.6% 해결을 하드웨어 측에서 보조)

**솔루션**:
- Costel 등 HW 제조사와 OTA 업데이트 채널 구축
- 펌웨어 버그 패치 무중단 배포 (심야 자동 업데이트)

**구현 주체**: HW 제조사(Costel) 협업

**✅ 수용 기준 (Acceptance Criteria)**:
- 원격 펌웨어 업데이트 성공률 ≥ 95% (실패 시 롤백 가능)
- 심야 무중단 배포 중 충전 세션 중단 0건

---

#### Feature 11: 충전 완료 자동 알림 + 이동 유예 시간

**근거**: 일반 혹평(7.3%)에 포함된 충전 종료 후 경험 개선 요구 (정량 근거가 약해 P2 후순위)

**솔루션**:
- 충전 완료 5분 전 푸시 알림
- 완료 후 10분 유예 시간 (추가 과금 없음)

**✅ 수용 기준 (Acceptance Criteria)**:
- 충전 완료 후 알림 도달률 ≥ 98%
- 충전 완료 시점~알림 수신 지연 ≤ 10초
- 사용자 알림 수신 동의율 ≥ 70%

## 4. 우선순위 매트릭스
